**无法上传文件到飞书云文档**

错误：400 - params error (错误代码：1061002)
```

#### 2. Drive API v1 - files/upload

URL: https://open.feishu.cn/open-apis/drive/v1/files/upload

URL: https://open.feishu.cn/open-apis/drive/medias/upload

#### 4. Drive API v1 - files (创建文件元数据)
```
URL: https://open.feishu.cn/open-apis/drive/v1/files

---

## 🔍 问题分析

### 可能的原因

1. **API 版本不正确**
   - 飞书可能有多个 API 版本（v1, v2 等）
   - 当前使用的 endpoint 可能已废弃或不存在

2. **应用权限问题**
   - 虽然 token 能获取，但可能没有实际的上传权限
   - 需要检查应用是否真的开通了 `drive:file:upload` 权限

3. **请求格式问题**
   - 可能需要使用飞书官方 SDK
   - 或者需要特殊的请求头或参数格式

4. **租户限制**
   - 某些企业可能禁用了云文档 API
   - 或者需要额外的配置

## ✅ 解决方案

### 方案 1：使用飞书官方 SDK（推荐）

安装飞书官方 Node.js SDK：

```bash
npm install @lark-base-open/js-sdk
```

使用 SDK 上传文件：

```javascript
const { Client } = require('@lark-base-open/js-sdk');

const client = new Client({
    appId: 'cli_a90abd1ceff99cc4',
    appSecret: 'oyvAMYAyZnYf76CKMRixbgcnynTO6kPB'
});

async function uploadFile(filePath) {
    const result = await client.drive.files.uploadAll({
        file: fs.createReadStream(filePath),
        folder_token: 'root'
    });
    return result;
}
```

### 方案 2：添加文本字段记录文件名（临时方案）

在飞书成果提交表中添加一个文本字段：

1. 打开成果提交表
2. 添加新列：**"附件文件名"**（文本类型）
3. 修改后端代码，将文件名写入这个字段

这样即使无法上传文件，也能记录文件名，后续可以手动上传。

### 方案 3：使用其他云存储

- 阿里云 OSS
- 腾讯云 COS
- 七牛云
- 本地文件服务器

上传后，将文件 URL 写入飞书表格。
