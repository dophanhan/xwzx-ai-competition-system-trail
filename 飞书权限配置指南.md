# 📋 飞书应用权限配置指南

## ❌ 当前问题

API 调用返回 400 错误，错误代码：`99991672`

**错误信息：**
```
Access denied. One of the following scopes is required: [bitable:app, base:record:create]
应用尚未开通所需的应用身份权限
```

## ✅ 解决方案

### 步骤 1：打开飞书开放平台

访问：https://open.feishu.cn/app/cli_a9283c06df79dcda/auth

或直接申请权限链接：
```
https://open.feishu.cn/app/cli_a9283c06df79dcda/auth?q=bitable:app,base:record:create&op_from=openapi&token_type=tenant
```

### 步骤 2：申请必需权限

需要申请以下**两个权限**：

#### 1️⃣ bitable:app（多维表格应用权限）
- **权限名称**：获取或更新多维表格记录
- **权限描述**：允许应用读取和写入多维表格数据
- **申请方式**：
  1. 应用详情页 → 权限管理
  2. 搜索「多维表格」或「bitable」
  3. 勾选「获取或更新多维表格记录」
  4. 点击「申请权限」

#### 2️⃣ base:record:create（创建记录权限）
- **权限名称**：创建多维表格记录
- **权限描述**：允许应用在多维表格中创建新记录
- **申请方式**：
  1. 应用详情页 → 权限管理
  2. 搜索「创建记录」或「record:create」
  3. 勾选「创建多维表格记录」
  4. 点击「申请权限」

### 步骤 3：等待审批

- 提交申请后，需要**企业管理员审批**
- 通常**立即生效**，最长不超过 24 小时
- 审批通过后，权限状态会变为「已开通」

### 步骤 4：验证权限

权限开通后，重新运行测试：

```bash
cd /home/admin/openclaw/workspace/ai_competition_system/ai_competition_system/backend
node debug_feishu.js
```

如果显示「✅ 创建成功！」，说明权限已生效。

## ⚠️ 常见问题

### Q1: 申请后仍然报错？
- 检查是否已等待审批通过
- 检查应用 ID 是否正确（cli_a9283c06df79dcda）
- 重启后端服务：`npm restart`

### Q2: 找不到权限申请入口？
- 确保你是应用的**管理员**或**开发者**
- 联系企业管理员协助申请

### Q3: 权限开通后需要重启服务吗？
- **不需要**，权限立即生效
- 但建议重启服务以确保配置刷新
