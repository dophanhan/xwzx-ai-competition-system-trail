# 🚨 阿里云函数计算部署问题排查

## 问题现象

日志显示：
```
[INFO] [rloader]: start to listen on: /tmp/r.s
```

这说明阿里云**没有执行 bootstrap 文件**，而是使用了默认的加载方式。

---

## 🔍 检查清单

### 1. 确认运行环境配置

登录阿里云函数计算控制台：
https://fcnext.console.aliyun.com/

进入服务 → 函数配置 → **运行环境**

**必须选择**：
- ✅ **自定义运行时** (Custom Runtime)
- ✅ **Node.js 16**
- ✅ **Debian 10**

**❌ 错误配置**：
- Node.js 16（非自定义运行时）
- Node.js 12
- Node.js 10

---

### 2. 确认启动命令

在函数配置页面，检查 **启动命令** 设置：

**正确配置**：
- 启动命令：**留空**（让系统自动执行 bootstrap）
- 或：`./bootstrap`

**❌ 错误配置**：
- `node app.js`
- `npm start`
- 任何其他命令

---

### 3. 确认代码包结构

上传的代码包解压后应该是：

```
/code/
├── bootstrap       # 👈 必须在根目录
├── app.js
├── package.json
└── node_modules/
```

**验证方法**：
1. 在阿里云控制台点击"代码管理"
2. 点击"在线编辑"
3. 查看文件列表，确认 `bootstrap` 在根目录

---

### 4. 确认 bootstrap 文件权限

bootstrap 文件必须有**执行权限**：

```bash
chmod +x bootstrap
```

在阿里云控制台的"在线编辑"中：
1. 点击 bootstrap 文件
2. 查看文件属性
3. 确认有执行权限（x）

---

## 🛠️ 解决方案

### 方案 A：重新配置运行环境

1. 登录阿里云：https://fcnext.console.aliyun.com/
2. 进入服务：ai-compfunction（杭州区域）
3. 点击"**函数配置**"
4. 点击"**编辑**"
5. 运行环境选择：**自定义运行时 Node.js 16**
6. 启动命令：**留空**
7. 点击"**保存**"
8. 重新上传代码包：backend-deploy-v11.zip
9. 点击"**部署**"

---

### 方案 B：手动指定启动命令

如果方案 A 不行，尝试手动指定：

1. 进入函数配置
2. 点击"编辑"
3. 运行环境：**自定义运行时 Node.js 16**
4. **启动命令**：`/code/bootstrap`
5. 保存并重新部署

---

### 方案 C：删除函数重新创建

如果以上都不行：

1. **删除当前函数**
2. **重新创建函数**：
   - 地域：华东 1（杭州）
   - 运行环境：**自定义运行时 Node.js 16**
   - 代码上传方式：上传 ZIP 包
   - 上传文件：backend-deploy-v11.zip
   - 启动命令：**留空**
3. 配置环境变量
4. 保存并部署
