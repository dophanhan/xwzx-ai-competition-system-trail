# AI 大赛报名系统 - 部署检查清单

## 部署前检查

### 1. 后端配置

- [ ] 复制 `.env.example` 为 `.env`
- [ ] 配置 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET`
- [ ] 配置正确的表格 ID：
  - `FEISHU_REGISTRATION_TABLE=tbltDHBoAs72cFnh`
  - `FEISHU_SUBMISSION_TABLE=tblxYsQVComD9wRW`
- [ ] （可选）配置飞书机器人 Webhook
- [ ] 运行 `npm install` 安装依赖

### 2. 前端配置

- [ ] 编辑 `frontend/config.js`
- [ ] 开发环境：`BASE_URL = 'http://localhost:3000/api'`
- [ ] 生产环境：修改为实际后端地址

### 3. 飞书多维表格字段配置

**报名表表格字段：**
- [ ] 队伍名称（文本）
- [ ] 队伍人数（数字）
- [ ] 队员 1 姓名（文本）
- [ ] 队员 2 姓名（文本）
- [ ] 队员 3 姓名（文本）
- [ ] 提交时间（文本/日期）

**成果提交表格字段：**
- [ ] 队伍名称（文本）
- [ ] 成果名称（文本）
- [ ] 团队分工（文本）
- [ ] 成果说明（文本）
- [ ] 在线链接（URL）
- [ ] 上传文件（文本）
- [ ] 提交时间（文本/日期）

### 4. 飞书应用权限

确保飞书应用已开通以下权限：
- [ ] 获取租户 access_token (tenant_access_token)
- [ ] 多维表格数据读写 (bitable:app)
- [ ] 发送消息到群组 (im:message)

## 启动服务

### 启动后端

```bash
cd backend
npm start
```

检查日志：
- [ ] 看到 "服务器运行在 http://localhost:3000"
- [ ] 看到正确的表格 ID 显示
- [ ] 没有报错信息

### 启动前端

```bash
cd frontend
python -m http.server 8080
```

或使用其他静态服务器。

## 常见问题排查

### 后端启动失败

1. 检查 Node.js 版本：`node --version`（需要 16.x+）
2. 检查依赖是否安装：`ls node_modules`
3. 检查端口是否被占用：`lsof -i:3000`

### 飞书 API 调用失败

1. 检查 `.env` 文件中的凭证是否正确
2. 检查飞书应用权限是否已开通
3. 检查表格 ID 是否正确
4. 查看后端日志中的错误信息

### 前端无法连接后端

1. 检查 `config.js` 中的 `BASE_URL` 是否正确
2. 检查后端服务是否启动
3. 检查浏览器控制台是否有 CORS 错误
4. 确保前后端在同一网络环境

## 生产环境部署

### 后端部署

1. 使用 PM2 管理进程：
   ```bash
   npm install -g pm2
   pm2 start app.js --name ai-competition
   pm2 save
   ```

2. 配置 Nginx 反向代理（可选）

3. 配置 HTTPS（推荐）

### 前端部署

1. 使用静态文件托管服务
2. 或使用 Nginx/Apache 托管
3. 修改 `config.js` 中的 `BASE_URL` 为生产地址
