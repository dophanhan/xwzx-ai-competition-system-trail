# 🐛 页面报错修复 - 2026-03-30

## 问题描述

用户在成果提交页面遇到 JavaScript 错误：

```
Uncaught TypeError: Cannot read properties of null (reading 'addEventListener')
    at submission.js:89:41

Uncaught (in promise) TypeError: Cannot set properties of null (setting 'innerHTML')
    at loadTeams (submission.js:57:32)
```

## 根本原因

代码在访问 DOM 元素时没有进行空值检查，当元素不存在或尚未加载时，`getElementById()` 返回 `null`，导致后续操作抛出异常。

### 受影响的文件
1. `frontend/submission.js` - 成果提交页面
2. `frontend/registration.js` - 报名页面

### 具体问题点

#### submission.js
1. **loadTeams()** - 未检查 `loadingTeams` 和 `teamName` 元素
2. **setupCharCount()** - 未检查输入框和计数显示元素
3. **DOMContentLoaded** - 未检查 `submission-form` 表单
4. **setupFileUpload()** - 未检查文件上传区域和输入框
5. **updateFileList()** - 未检查文件列表元素
6. **showError/showSuccess** - 未检查消息元素

#### registration.js
1. **DOMContentLoaded** - 未检查 `addMemberBtn` 和 `registration-form`

## 修复方案

### 1. 添加 DOM 元素存在性检查

在所有使用 `getElementById()` 的地方添加空值检查：

```javascript
// 修复前
const formEl = document.getElementById('submission-form');
formEl.addEventListener('submit', handleSubmit);

// 修复后
const formEl = document.getElementById('submission-form');
if (!formEl) {
    console.error('找不到 submission-form 表单元素');
    return;
}
formEl.addEventListener('submit', handleSubmit);
```

### 2. 增强错误处理

为消息显示函数添加降级处理：

```javascript
function showError(message) {
    const errorDiv = document.getElementById('error-message');
    if (errorDiv) {
        errorDiv.textContent = message;
        errorDiv.classList.add('active');
    } else {
        console.error('找不到错误消息元素');
        alert(message); // 降级处理
    }
}
```

### 3. 添加调试日志

在关键位置添加 `console.log` 帮助排查问题：

```javascript
document.addEventListener('DOMContentLoaded', function() {
    console.log('页面加载完成');
    console.log('表单元素:', formEl);
    console.log('表单提交事件已绑定');
});
```
